# GitHub Reviews

Quando usar:

- PR com `changes requested`.
- Comentários técnicos pendentes.
- Threads não resolvidas.

Fluxo esperado:

1. Listar apenas comentários acionáveis.
2. Agrupar por arquivo e impacto.
3. Aplicar mudanças mínimas no escopo do PR.
4. Responder cada thread com o que foi alterado.
5. Sinalizar riscos e pontos que exigem validação adicional.

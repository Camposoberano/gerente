# Release Ops

Quando usar:

- Preparar entrega de versão.
- Publicar release notes.
- Organizar fluxo de merge para produção.

Fluxo esperado:

1. Definir versão e escopo (semver).
2. Consolidar changelog por impacto.
3. Verificar pré-condições (CI, QA, dependências).
4. Preparar rollback objetivo.
5. Entregar passos de publicação em ordem executável.

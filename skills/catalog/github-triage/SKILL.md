# GitHub Triage

Quando usar:

- Backlog desorganizado.
- Muitas issues sem prioridade.
- PRs parados sem responsável.

Fluxo esperado:

1. Classificar cada item por criticidade (`crítico`, `alto`, `médio`, `baixo`).
2. Sugerir dono e prazo para cada item.
3. Aplicar labels e milestones consistentes.
4. Entregar checklist das ações imediatas (24h) e próximas ações (7 dias).

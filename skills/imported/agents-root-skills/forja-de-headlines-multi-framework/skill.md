# Forja de Headlines Multi-Framework

Esta skill não entrega UMA lista de headlines — entrega **o mesmo conceito passado por diferentes frameworks comprovados**, para você ver qual arquitetura ressoa mais com seu público antes de rodar um teste A/B cheio.

## A lógica

A maioria das skills de headline aplica um único framework (geralmente AIDA) e varia tons. O problema é que AIDA é bom pra venda direta, péssimo pra awareness. PAS mata em público consciente da dor, naufraga em público frio. Cada framework tem um "ponto doce" de aplicação — e você precisa testar a arquitetura, não só a frase.

## Frameworks que a skill usa

| Framework | Quando brilha | Quando falha |
|-----------|---------------|--------------|
| AIDA (Atenção-Interesse-Desejo-Ação) | Ofertas claras para público morno/quente | Público frio que ainda não reconheceu a dor |
| PAS (Problema-Agitação-Solução) | Público que já sabe que tem o problema | Venda aspiracional, produtos "quero ter" |
| 4Us (Útil-Urgente-Único-Ultra-específico) | Anúncios pagos e assuntos de email | Headlines longas de LP |
| BAB (Before-After-Bridge) | Transformação tangível, antes/depois claro | Produtos de entretenimento ou prazer |
| Curiosity Gap (Lacuna de Curiosidade) | Redes sociais e tráfego orgânico | Landing pages (alta rejeição) |

## O Prompt

```
Você é um diretor de criação de copy com passagem por agência de performance e casa de infoproduto. Sua especialidade não é só escrever headlines — é saber QUAL arquitetura de headline funciona pra cada combinação de produto + temperatura de público + canal.

Vou te dar um contexto. Você vai entregar headlines usando 5 frameworks diferentes, e no final recomendar qual framework apostar primeiro.

**CONTEXTO:**
- Produto/oferta: [descreva]
- Público: [perfil demográfico + psicográfico]
- Temperatura do público: [frio / morno / quente]
- Canal onde a headline vai rodar: [LP, Meta Ads, Google Ads, email, orgânico, outro]
- Principal dor reconhecida pelo público: [a dor que ELE já nomeia]
- Resultado tangível que o produto entrega: [número, prazo, transformação]
- Prova disponível: [cases, números, autoridade, depoimentos]
- Tom da marca: [uma linha]
- Palavras-veto: [termos que não podem aparecer — marcas, clichês, gírias, etc.]

**ENTREGA:**

Para cada um dos 5 frameworks abaixo, gere 3 headlines + 1 subheadline de apoio + 1 linha de justificativa técnica (por que essa arquitetura cabe aqui).

1. **AIDA** — para ofertas com benefício claro e público que já reconhece valor
2. **PAS** — para quando a dor é o gatilho principal
3. **4Us** — para headlines curtas de alta fricção (ads, assuntos)
4. **BAB** — para transformação tangível
5. **Curiosity Gap** — para cortar ruído em feed ou inbox

Depois das 15 headlines, entregue:

**DIAGNÓSTICO DO CASO**
- Qual framework melhor serve esse contexto específico (e por quê em 2 frases)
- Qual framework eu provavelmente NÃO deveria testar nesse canal (e por quê)
- Se houver risco de violação de política (Meta, Google), sinalize

**SUGESTÃO DE TESTE**
Plano de batalha de 3 headlines pra subir primeiro:
- Headline 1 (framework principal)
- Headline 2 (variação do mesmo framework, ângulo oposto)
- Headline 3 (framework secundário como controle)

Por fim, liste 3 palavras-gatilho específicas desse nicho que devem aparecer em pelo menos uma das variações.
```

## Como usar bem

1. **Preencha "palavras-veto"** — é onde a maioria erra. Se você não disser que odeia "revolucionário", "game changer" e "descubra o segredo", vai aparecer.
2. **Separe temperatura de público** — headline pra lead frio e lead quente são animais diferentes; não misture.
3. **Rode dois canais no mesmo contexto** — gere uma vez pra LP e outra pra ad. Compare qual framework venceu em cada canal. Isso vira repertório.

## Sinais de que a skill funcionou

- Você consegue explicar em uma frase POR QUE escolheu determinada headline
- As 3 headlines do plano de batalha são arquiteturalmente diferentes entre si (não variações cosméticas)
- Pelo menos uma quebra uma convenção do seu nicho

## Erros comuns de quem usa

- Pedir "muitas headlines" em vez de "headlines diferentes" — mais não é melhor, é ruído
- Não preencher o campo de prova — headline sem prova vira promessa vazia
- Usar pra título de YouTube (existe skill específica pra isso)

---
**Quando aumentar a aposta:** Para produto lançando do zero, peça também 2 headlines "contrárias" — no ângulo que a concorrência não usa. Ângulo virgem geralmente converte mais no primeiro mês que headline "best practice".
